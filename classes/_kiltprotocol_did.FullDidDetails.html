constructor

  • new FullDidDetails(creationDetails: DidConstructorDetails & { identifier: string }): FullDidDetails

  • Overrides DidDetails.constructor

    • Defined in did/src/DidDetails/FullDidDetails.ts:62

    Create an instance of FullDidDetails with the provided details. This is not to be used to create new DIDs, but it should only be used to serialize-deserialize full DID information to and from storage. Creating an instance of a full DID in this way without writing the information on the blockchain, will render the DID useless.

    Parameters

    • creationDetails: DidConstructorDetails & { identifier: string }

      The creation details.

    Returns FullDidDetails

authorizeExtrinsic

  • authorizeExtrinsic(extrinsic: Extrinsic, signer: KeystoreSigner<any>, submitterAccount: string, signingOptions?: { keySelection?: DidKeySelectionCallback<DidVerificationKey>; txCounter?: BN }): Promise<SubmittableExtrinsic>
    • Defined in did/src/DidDetails/FullDidDetails.ts:176

    Signs and returns the provided unsigned extrinsic with the right DID key, if present. Otherwise, it will throw an error.

    Parameters

    • extrinsic: Extrinsic

      The unsigned extrinsic to sign.

    • signer: KeystoreSigner<any>

      The keystore signer to use.

    • submitterAccount: string

      The KILT account to bind the DID operation to (to avoid MitM and replay attacks).

    • signingOptions: { keySelection?: DidKeySelectionCallback<DidVerificationKey>; txCounter?: BN } = {}

      The signing options.

      • Optional keySelection?: DidKeySelectionCallback<DidVerificationKey>

        The optional key selection logic, to choose the key among the set of allowed keys. By default it takes the first key from the set of valid keys.

      • Optional txCounter?: BN

        The optional DID nonce to include in the operation signatures. By default, it uses the next value of the nonce stored on chain.

    Returns Promise<SubmittableExtrinsic>

    The DID-signed submittable extrinsic.
