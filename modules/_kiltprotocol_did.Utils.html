checkServiceEndpointSizeConstraints

  • checkServiceEndpointSizeConstraints(api: ApiPromise, endpoint: DidServiceEndpoint): [boolean, Error[] | undefined]
    • Defined in did/src/Did.utils.ts:385

    Performs size checks on service endpoint data, making sure that the following conditions are met:

    • The endpoint.id is at most 50 ASCII characters long.
    • The endpoint.types array has at most 1 service type, with a value that is at most 50 ASCII characters long.
    • The endpoint.urls array has at most 1 URL, with a value that is at most 200 ASCII characters long.

    Parameters

    • api: ApiPromise

      An api instance required for reading up-to-date size contraints from the blockchain runtime.

    • endpoint: DidServiceEndpoint

      A service endpoint object to check.

    Returns [boolean, Error[] | undefined]

    Validation result and errors, if any.
